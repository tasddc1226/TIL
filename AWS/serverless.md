# Serverless
> Section 19: 솔루션 설계자 관점의 서버리스

## Serverless가 무엇인가?
- 서버가 없다는 것은 아니고 관리할 필요가 없다는 뜻이다.
- 코드 및 함수를 배치하는 것이다.
- 처음 서버리스가 개발된 것은 AWS Lambda 였다.

## AWS Lambda가 무엇이고 무슨 도움을 주는지?
> AWS EC2와 비교를 통해서 살펴보기

### AWS EC2
- 클라우드의 가상 서버
- 제한된 RAM과 CPU를 사용하므로 **프로비저닝이** 필요함.
- 지속적으로 실행해야 하므로 최적화를 위해서는 효율적으로 시작하고 중단해야 한다.
- 오토 스케일링 그룹으로 스케일링할 수 있는데 자동으로 서버를 추가 제거하는 작업이 필요함.

> 프로비저닝?
> - 프로비저닝은 사용자의 요구에 맞게 시스템 자원을 할당, 배치, 배포해 두었다가 필요 시 시스템을 즉시 사용할 수 있는 상태로 미리 준비해 두는 것을 말한다. 

### AWS Lambda
- 람다는 가상의 함수이다.
- 관리할 서버 없이 코드를 프로비저닝하면 함수가 실행
- 제한 시간이 있어서 실행 시간이 짧다고 하지만, 최대 15분은 짧지 않다
- On-demand로 실행되어진다.
    - 이는 Lambda를 사용하지 않으면 람다 함수가 실행 X
- 스케일링이 자동화된다.
    - 더 많은 람다 함수가 필요하다면 AWS가 알아서 프로비저닝을 통해 람다 함수를 늘려준다.

### AWS Lambda의 장점?
- 가격 책정이 매우 쉽다.
  - 수신하는 요청의 횟수에 따라 과금
  - 또한 코드를 실행하면 정말 저렴하다!

- 다양한 AWS 서비스와 통합될 수 있음.
- 또한 다양한 프로그래밍 언어를 사용할 수 있어서 매우 자유로운 편에 속함.
- `CloudWatch`를 통한 모니터링 통합도 쉽다.
- 만약 함수의 RAM을 증가시킨다면 CPU 및 네트워크 품질과 성능도 함께 향상
- 함수당 최대 10GB의 램을 프로비저닝 가능하다.

#### Tip
> 람다를 처음 만들고 정말 테스트용이 아니고서는 128MB 그대로 사용하지 말고 1536MB로 세팅해서 쓰는걸 추천
