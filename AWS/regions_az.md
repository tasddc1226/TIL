# AWS regions and AZ(Availability Zones)

## what is region?
- AWS의 리전은 전세계에 걸쳐서 존재한다.
- 이는 데이터 센터의 집합을 의미한다.
- 이름으로는 us-east-1, eu-west-3 ... 이 될 수 있다.
- 대부분의 서비스들은 특정 리전(지역)에 연결되어 국한되어진다.
- 각 리전끼리는 독립적이어서 이전에 사용하던 리전에서 다른 리전으로 옮기려고 한다면 서비스를 처음 사용하는 것과 마찬가지이다.

### Q. how to choose an AWS Region?
- AWS 리전을 어떻게 선택할까요?

> 상황에 따라서 다르다!

- AWS 리전 선택에 영향을 미칠 수 있는 몇가지 요인들..

1. 법률 준수

    - 어떤 정부들은 Application을 배포하게 될 대상 국가 내에 데이터가 보관되어지기를 원한다.

2. 지연 시간(reduced latency)

    - 사용자가 미국인이 많다면 미국쪽의 리전을 사용하는 것이 맞다!
    - Application 역시 사용자와 가까이 있을 수 있도록해야 지연 시간이 줄어들기 때문이다.

3. 모든 리전이 모든 서비스를 가지고 있지는 않다.

    - 일부 리전에는 서비스가 없을 수도 있다.
    - 따라서, Application에 특정 AWS의 서비스를 활용중이라면, 그 리전의 해당 서비스가 존재하는지를 확인해야 한다.

4. 요금

    - 리전마다 요금이 다르다.


### AWS Availability Zones (가용 영역)
> 가용 영역은 리전 내에 존재한다.
> 각 리전은 가용 영역이 존재하고 대게는 3개부터 최소 2개부터 최대 6개까지 있다.

- 각각의 가용 영역(AZ)은 하나 또는 그 이상의 데이터 센터가 있을 수 있다.
- AZ들은 재난 발생에 대비하여 서로 분리되어 있다는 것은 확실하다.
- 데이터 센터와 가용 영역들은 **높은 대역폭의 초저지연 네트워킹**으로 연결되어 리전 형성


### 전송 지점 (Edge Locations)
> AWS에 대해 꼭 알고 있어야 할 점으로 전송 지점이 있다.

- 많은 도시에 많은 전송 지점을 가지고 있기 때문에 최소 지연 시간으로 최종 사용자에게 컨텐츠를 전달해준다는 것에 유용하다.


#### IAM과 EC2의 차이
- IAM 같은 경우에는 **글로벌**서비스이기 때문에 리전이 불필요하다.
- 반면 EC2는 리전 서비스이므로 선택한 지역이 노출되어진다.
- 따라서 우측 상단에서 리전 서비스인지 아닌지 알 수 있다.
